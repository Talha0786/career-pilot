# 001 — Monorepo scaffold, tooling, boundary lint

**Status:** DONE | **Milestone:** M2

## Objective
Stand up the pnpm + Turborepo workspace with strict TypeScript, and make the Clean Architecture dependency rule a *build failure* rather than a convention.

## Files affected
- `package.json`, `pnpm-workspace.yaml`, `turbo.json`, `tsconfig.base.json`, `.nvmrc`, `.gitignore`, `.env.example`
- `packages/config/{eslint,tsconfig,prettier}`

## Dependencies
None.

## Acceptance criteria
- [ ] `pnpm install` succeeds from a clean clone
- [ ] `pnpm typecheck` and `pnpm lint` run across the workspace
- [ ] Importing `infrastructure` from `domain` **fails lint** (verified on a deliberately-broken branch)
- [ ] `.env.example` documents every variable; no secrets committed
- [ ] `tsconfig` is `strict: true` with `noUncheckedIndexedAccess`

## Test plan
Negative test: a scratch branch adds a forbidden import; `pnpm lint` must exit non-zero. This is the test that makes the architecture real.

## Notes
Only create packages the M2 slice needs. Empty directories are not architecture.
