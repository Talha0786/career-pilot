# 016 — Wire PostgresBudgetStore's advisory lock into GuardedLlmPort

**Status:** DONE | **Milestone:** M2

## Objective
`GuardedLlmPort.embed()` currently calls `store.getMonthlySpend()` and
`store.recordInvocation()` directly, unlocked. Task 015 proved the lock
primitive works in isolation; this task makes the guard actually use it.

## Files affected
- `packages/application/src/ports/budget-guard.ts`
- `packages/infrastructure/src/llm/postgres-budget-store.ts` (BudgetStore
  interface gains `withUserBudgetLock` as a required method, or the guard
  takes it as a separate optional dependency — decide during implementation)

## Dependencies
015

## Acceptance criteria
- [x] `GuardedLlmPort.embed()` wraps its check+dispatch+record in
      `withUserBudgetLock` when the store supports it
- [x] `InMemoryBudgetStore` (test fake) gets an equivalent single-flight lock
      so unit tests can exercise the same guarantee without Postgres
- [x] A guard-level (not store-level) integration test: N concurrent
      `guard.embed()` calls against a tight budget, exact-count assertion

## Design note
`withUserBudgetLock` is optional on `BudgetStore` — a store without it just
runs the guard unlocked (unchanged prior behavior). The lock spans the
**entire** check-dispatch-record sequence, including the outbound LLM call,
not just the DB bookkeeping — a deliberate M2-scale simplification traded
against per-user throughput (concurrent embeds for the SAME user are fully
serialized; different users never block each other). Narrowing the critical
section to just the check+reservation is a future refinement, not required
by this task's acceptance criteria.

## Test plan — done
- Unit (`packages/application/test/unit/budget-guard.test.ts`): 10 truly
  concurrent (`Promise.all`) `guard.embed()` calls against a budget sized
  for exactly 2, using `InMemoryBudgetStore`'s new single-flight lock —
  exact count, not "usually right." Plus a same-shape test proving two
  DIFFERENT users' calls do NOT serialize against each other.
- Integration (`packages/infrastructure/test/integration/budget-lock.test.ts`,
  new describe block): the task-015 proof repeated one layer up — 20
  concurrent `GuardedLlmPort.embed()` calls, each on its own real Postgres
  connection, against a real `PostgresBudgetStore`; exactly 3 succeed, every
  run.
