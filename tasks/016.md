# 016 — Wire PostgresBudgetStore's advisory lock into GuardedLlmPort

**Status:** TODO | **Milestone:** M2

## Objective
`GuardedLlmPort.embed()` currently calls `store.getMonthlySpend()` and
`store.recordInvocation()` directly, unlocked. Task 015 proved the lock
primitive works in isolation; this task makes the guard actually use it.

## Files affected
- `packages/application/src/ports/budget-guard.ts`
- `packages/infrastructure/src/llm/postgres-budget-store.ts` (BudgetStore
  interface gains `withUserBudgetLock` as a required method, or the guard
  takes it as a separate optional dependency — decide during implementation)

## Dependencies
015

## Acceptance criteria
- [ ] `GuardedLlmPort.embed()` wraps its check+dispatch+record in
      `withUserBudgetLock` when the store supports it
- [ ] `InMemoryBudgetStore` (test fake) gets an equivalent single-flight lock
      so unit tests can exercise the same guarantee without Postgres
- [ ] A guard-level (not store-level) integration test: N concurrent
      `guard.embed()` calls against a tight budget, exact-count assertion
