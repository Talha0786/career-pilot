# 032 — Connector health + chaos test

**Status:** TODO | **Milestone:** M4

## Objective
Close the loop on connector isolation (design's `connector_configs.health` enum) and prove it under real failure, matching task 014's chaos-testing standard.

## Files affected
- `packages/application/src/discovery/commands/update-connector-health.ts` (healthy/degraded/disabled transitions based on consecutive `ingestion_runs` failures)
- `apps/api/src/routes/connectors.ts` (`GET /connectors`, `PATCH /connectors/:id` enable/config/BYO-key, per `docs/README.md`'s documented route shape)
- `e2e/chaos/connector-failure.spec.ts` (new chaos test: one connector's fetch deliberately throws on every call; assert its health flips to `degraded` after N consecutive failures, other connectors' scheduled runs are unaffected, and it flips back to `healthy` after a subsequent success)

## Dependencies
028, 029, 014

## Acceptance criteria
- [ ] Health transitions are driven by real consecutive-failure counting against `ingestion_runs`, not a mocked signal
- [ ] Chaos test uses a real broken connector implementation (throws for real), not a simulated flag — same standard as task 014's real `SIGKILL`
- [ ] `PATCH /connectors/:id` for BYO-key config never returns the stored key value in any response body (write-only field)

## Test plan
The chaos e2e test itself is the primary proof — run it and record pass/fail with actual health-state transitions observed, same evidence bar as task 014's `worker-kill.spec.ts`.
