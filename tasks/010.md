# 010 — Worker: BullMQ + idempotent embed handler

**Status:** TODO | **Milestone:** M2

## Objective
The worker process: consume `JobPosted`, embed via `LlmPort`, persist, notify. The reference implementation every future handler copies.

## Files affected
- `apps/worker/src/{main.ts,handlers/embed-job-posting.handler.ts}`

## Dependencies
008, 009

## Acceptance criteria
- [ ] Handler is **idempotent** by `(jobPostingId, model)` — ADR-007 makes this mandatory for every handler, forever
- [ ] Redelivering the same event yields **one** embedding and **one** `ai_invocation` row
- [ ] Failure → `embedding_status = failed`, error logged, retried with backoff; the job never crashes the worker
- [ ] On success publishes `job.embedded` to Redis pub/sub for the API's WebSocket fanout
- [ ] Structured logs (pino) with a correlation id propagated from the originating HTTP request

## Test plan
Integration: deliver the same event twice → assert single effect (the ADR-007 guarantee, tested).
Chaos: kill the worker mid-handler → on restart, zero lost events, zero duplicate rows.
