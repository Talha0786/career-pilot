# 031 — Class C reference adapter: SerpApi Google Jobs (BYO key)

**Status:** DONE (fixture-verified; live canary pending `SERPAPI_KEY` secret) | **Milestone:** M4

## Objective
The reference BYO-key licensed-provider adapter (ADR-004 Class C). Proves the pattern for future Class C adapters (Mantiks/Bright Data/Coresignal, all post-1.0).

## Files affected
- `packages/connectors/class-c/serpapi-google-jobs/index.ts`
- `packages/connectors/class-c/serpapi-google-jobs/test/fixtures/*.json` (recorded SerpApi responses)
- `docs/connectors/class-c-serpapi.md` (setup doc: user obtains their own SerpApi key, where to enter it, what it costs — this is a user-facing "you need your own key" doc, not something this task can bypass)

## Dependencies
026, 028

## Acceptance criteria
- [ ] Adapter passes the shared contract test-kit against fixtures — **does not require a real SerpApi key to pass in default CI** (fixture-based, matching 028's pattern)
- [ ] Missing/invalid API key surfaces as a typed config error at connector-enable time, not a runtime crash mid-ingestion
- [ ] Nightly canary job (opt-in, requires a real key as a GitHub secret **the user must add** — flagged explicitly, not assumed present) exercises one real live call
- [ ] Costs and liability posture documented per ADR-004 (user's contract with SerpApi, not CareerPilot's)

## Test plan
Unit/integration: fixture-based contract test-kit run, config-error path test. Nightly live canary is best-effort and skips gracefully if no key is configured (same skip-if-unreachable pattern as the existing Ollama nightly test, task 014).

## Known gap (flagged in advance)
This adapter cannot be verified end-to-end against the real SerpApi service without a paid API key, which only the user can provide (BYO-key by design — ADR-004). The nightly canary will report itself as skipped, not failed, until a `SERPAPI_KEY` secret is added to the repo.

## Test plan actually executed

Everything verifiable without a paid key was verified for real, and the one thing that genuinely requires a key is isolated to a single, clearly-skipping nightly test (not blocking this task or the track).

- **What WAS confirmed against the real live SerpApi endpoint** (no key required for this part): `GET https://serpapi.com/search.json?engine=google_jobs&q=...&api_key=invalid-key-for-probing` from within this task's Docker verification container returned **HTTP 401** with body `{"error": "Invalid API key. Your API key should be here: https://serpapi.com/manage-api-key"}` — exactly the shape `createSerpapiGoogleJobsConnector`'s `auth_error` handling is coded against and tested against (`index.test.ts`'s "surfaces a real, live-confirmed 401 shape" test literally reproduces that exact response body).
- `packages/connectors/src/class-c/serpapi-google-jobs/test/fixtures/search.json` is **hand-authored, not live-recorded** (disclosed in both the test file's header comment and here) — matches SerpApi's published Google Jobs API documentation as closely as a fixture authored without a paid key can. This is the ONE fixture across all of M4's connectors (028/030/031 combined) that isn't either live-recorded or algorithmically generated from user input — flagged consistently with task 028's identical USAJobs gap, not hidden.
- 6 unit tests, all passing: contract test-kit conformance (class C), full field normalization from the fixture (title/company/location/remote/salary/descriptionMd/url — url resolved from `apply_options[0].link`, falling back to `share_link`), the acceptance criterion's literal test ("missing/invalid API key surfaces as a typed config error at connector-enable time" — proven via `configSchema.safeParse` rejecting an empty key BEFORE any network call), the live-confirmed 401→auth_error mapping, a 429→rate_limited mapping, and the relative-posted_at-text parser (parses "2 days ago" into an approximate `Date`, returns `null` rather than guessing for unrecognized text like "sometime last quarter").
- **Nightly canary** (`packages/connectors/test/nightly/serpapi-canary.test.ts`) — actually run in this session: reports `Skipping: SERPAPI_KEY not configured` and passes (1/1), proving the skip path itself works cleanly rather than hanging or crashing. `.github/workflows/nightly.yml`'s `connectors-live-canary` job now passes `SERPAPI_KEY: ${{ secrets.SERPAPI_KEY }}` through — becomes a real live call automatically once the user adds that repo secret, no code change required.
- `docs/connectors/class-c-serpapi.md` documents setup (where the key goes — `credentials_ref`, never the `config` JSONB), SerpApi's own pricing/free-tier posture, and the liability framing from ADR-004 (SerpApi's ToS, not CareerPilot's).
- Full regression: `turbo run typecheck` 10/10, `eslint packages/connectors/src` 0 errors, full unit suite **243/243** (6 new for this task).

This task is marked DONE rather than blocked because everything short of "make a real paid API call" was actually built and verified — matching the instruction to build what CAN be fixture-verified and move the track forward rather than stalling on a credential only the end user can supply.
