# 031 — Class C reference adapter: SerpApi Google Jobs (BYO key)

**Status:** TODO | **Milestone:** M4

## Objective
The reference BYO-key licensed-provider adapter (ADR-004 Class C). Proves the pattern for future Class C adapters (Mantiks/Bright Data/Coresignal, all post-1.0).

## Files affected
- `packages/connectors/class-c/serpapi-google-jobs/index.ts`
- `packages/connectors/class-c/serpapi-google-jobs/test/fixtures/*.json` (recorded SerpApi responses)
- `docs/connectors/class-c-serpapi.md` (setup doc: user obtains their own SerpApi key, where to enter it, what it costs — this is a user-facing "you need your own key" doc, not something this task can bypass)

## Dependencies
026, 028

## Acceptance criteria
- [ ] Adapter passes the shared contract test-kit against fixtures — **does not require a real SerpApi key to pass in default CI** (fixture-based, matching 028's pattern)
- [ ] Missing/invalid API key surfaces as a typed config error at connector-enable time, not a runtime crash mid-ingestion
- [ ] Nightly canary job (opt-in, requires a real key as a GitHub secret **the user must add** — flagged explicitly, not assumed present) exercises one real live call
- [ ] Costs and liability posture documented per ADR-004 (user's contract with SerpApi, not CareerPilot's)

## Test plan
Unit/integration: fixture-based contract test-kit run, config-error path test. Nightly live canary is best-effort and skips gracefully if no key is configured (same skip-if-unreachable pattern as the existing Ollama nightly test, task 014).

## Known gap (flagged in advance)
This adapter cannot be verified end-to-end against the real SerpApi service without a paid API key, which only the user can provide (BYO-key by design — ADR-004). The nightly canary will report itself as skipped, not failed, until a `SERPAPI_KEY` secret is added to the repo.
