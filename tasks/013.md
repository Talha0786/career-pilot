# 013 — Docker Compose stack

**Status:** TODO | **Milestone:** M2

## Objective
One command from clean clone to a running application. If this is hard, the project is dead on arrival for self-hosters.

## Files affected
- `docker-compose.yml`, `compose.dev.yml`, `Dockerfile` per app, `.dockerignore`

## Dependencies
011, 012

## Acceptance criteria
- [ ] `cp .env.example .env && docker compose up` → healthy stack, **zero** manual steps
- [ ] One-shot `migrate` service runs before `api` (`depends_on: service_completed_successfully`)
- [ ] Healthchecks gate startup ordering; no race on first boot
- [ ] Non-root users; multi-stage builds; pinned base image digests
- [ ] `compose.dev.yml` runs Postgres + Redis only (apps via `pnpm dev`)
- [ ] Boots with **no LLM key** (local Ollama default, ADR-006)

## Test plan
CI: `docker compose up -d`, poll `/readyz` until green, run the e2e suite against it, tear down.
Manual: clean-machine install by someone who didn't build it (M2 acceptance gate).
