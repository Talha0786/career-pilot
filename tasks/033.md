# 033 — Real CostEstimator (per-model pricing table)

**Status:** TODO | **Milestone:** M5

## Objective
Replace the coarse inline `CostEstimator` stub in `apps/worker/src/main.ts` (comment: *"Cost estimation is deliberately coarse for M2 — real per-provider pricing tables are an M5 concern (ADR-006)"*) with a real implementation in `packages/infrastructure` driven by a per-model pricing table (input/output token rates, embedding-per-1k rates), so `GuardedLlmPort`'s pre-check is accurate enough to actually hard-stop at budget for M5's higher-volume matching/tailoring calls.

## Files affected
- `packages/infrastructure/src/llm/cost-estimator.ts` (new) — `TieredCostEstimator implements CostEstimator` (interface already defined in `packages/application/src/ports/budget-guard.ts` — do not change the interface).
- `packages/infrastructure/src/llm/pricing-table.ts` (new) — static table keyed by model id (e.g. `nomic-embed-text`, `gpt-4o-mini`, `gpt-4o`, local-Ollama entries priced at $0), `{ inputPer1kUsd, outputPer1kUsd, embedPer1kUsd }`; include a documented fallback rate for unknown model ids (log a warning, don't throw — an unpriced model must never crash a request, but must never be silently free either — use a conservative non-zero default).
- `apps/worker/src/main.ts` — replace the inline stub object with `new TieredCostEstimator(pricingTable)`.
- `apps/api/src/main.ts` — not wired yet (task 038 adds LLM wiring to the API); no change here.
- `packages/infrastructure/test/unit/cost-estimator.test.ts` (new).

## Dependencies
None (foundational; can start immediately).

## Acceptance criteria
- `TieredCostEstimator.estimate(req)` returns a cost in USD for both `EmbedRequest` and `CompleteRequest` shapes, proportional to token/char count and the request's model id.
- Unknown model id → conservative non-zero fallback rate + one warning log line (not a thrown error, not $0).
- Worker boots with the real estimator; no behavior regression in existing M2 chaos test (job-posted embed flow).

## Test plan
- Unit: table lookup for known models, fallback path for unknown model, zero-length input edge case.
- Existing `chaos` CI job (worker embed flow) must stay green — proves the swap didn't break the guarded embed path.
