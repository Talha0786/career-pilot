# 018 — UI: `@careerpilot/ui` component library + Tailwind restyle

**Status:** DONE | **Milestone:** M2

## Objective
`apps/web` currently renders with zero styling (plain unstyled HTML forms). Give CareerPilot a real, reusable component library — design tokens, a broad primitive set, accessible overlays — and restyle the three existing pages (login/register/board) on top of it. This is prep work for syncing a real design system to claude.ai/design, not a cosmetic pass.

## Files affected
- `packages/ui/{package.json,tsconfig.json,src/**}` — new package
- `apps/web/{package.json,postcss.config.mjs,src/app/globals.css,src/app/layout.tsx}`
- `apps/web/src/app/{login,register,board}/page.tsx` — restyled on top of `@careerpilot/ui`

## What was built
- `packages/ui` — new workspace package, consumed as raw `.tsx` source via `"main"/"types": "./src/index.ts"` (same pattern as `@careerpilot/contracts`), plus a `"./tokens.css"` export.
- `packages/ui/src/tokens.css` — Tailwind v4 CSS-first `@theme` (no `tailwind.config.js`): primary/neutral/success/warning/danger `oklch()` color scales, radius scale, shadow scale, `--font-sans`, plus light/dark semantic aliases.
- 22 components in `packages/ui/src/components/`: `Button`, `IconButton`, `Input`, `Textarea`, `Select` (Radix), `Checkbox` (Radix), `Switch` (Radix), `Badge`, `Avatar` (Radix), `Spinner`, `Divider`, `Card`/`CardHeader`/`CardTitle`/`CardDescription`/`CardContent`/`CardFooter`, `Alert`, `FormField`, `Dialog`/`SimpleDialog` (Radix), `DropdownMenu` (Radix), `Tabs` (Radix), `Tooltip` (Radix), `Toast`/`ToastProvider`/`useToast` (Radix), `Table` family, `KanbanColumn`/`KanbanCard`, `AppHeader`, `AuthShell`.
- `apps/web`: added `tailwindcss`/`@tailwindcss/postcss` deps, `postcss.config.mjs`, `src/app/globals.css` (imports `@careerpilot/ui/tokens.css` + a `@source` pointing at `packages/ui/src` so Tailwind's scanner sees classes used in the workspace package), wired into `layout.tsx`.
- Restyled `login/page.tsx`, `register/page.tsx` (both on `AuthShell`/`FormField`/`Input`/`Button`/`Alert`) and `board/page.tsx` (`AppHeader`/`Card`/`Textarea`/`KanbanColumn`/`KanbanCard`) — no raw unstyled `<input>`/`<button>` remaining.

## Bugs found and fixed
- `exactOptionalPropertyTypes: true` (from `tsconfig.base.json`) rejected several components that assigned possibly-`undefined` values to optional props directly (`Dialog.tsx`, `FormField.tsx`, `Select.tsx`). Fixed throughout via the conditional-spread pattern `{...(x !== undefined ? { x } : {})}`.
- `apps/web/src/app/board/page.tsx`'s `KanbanCard` `subtitle` prop is `string | undefined`, but `ApplicationCard.company` is `string | null` — a plain `!== undefined` guard didn't satisfy `exactOptionalPropertyTypes`. Fixed by guarding on truthiness (`card.company ? { subtitle: card.company } : {}`).
- `apps/web/Dockerfile` didn't `COPY` `packages/ui` into the `deps`/`build` stages, so `docker compose up -d --build web` failed with `Module not found: Can't resolve '@careerpilot/ui'`. Added the missing `COPY packages/ui/package.json` (deps stage) and `COPY packages/ui ./packages/ui` (build stage) lines, mirroring the existing pattern for `packages/infrastructure`.

They'll get their first live exercise whenever a future task's UI calls for them.
