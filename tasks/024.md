# 024 — Document engine: structured model → PDF/DOCX rendering

**Status:** TODO | **Milestone:** M3

## Objective
Render a `DocumentVersion`'s structured content into PDF and DOCX. Constrained to **2 templates** (roadmap's explicit scope guard against the "doc rendering rabbit hole").

## Files affected
- `packages/infrastructure/src/documents/render/{template-classic,template-modern}.ts`
- `packages/infrastructure/src/documents/render/pdf-renderer.ts`, `docx-renderer.ts`
- `packages/application/src/documents/commands/render-document.ts` (writes `rendered_pdf_key` per design §2, stores the artifact — reuse whatever object-storage approach the compose stack already provides, or local volume + signed API route if none exists yet)
- golden-file fixtures under `packages/infrastructure/test/fixtures/documents/`

## Dependencies
019, 021

## Acceptance criteria
- [ ] Round-trip: structured content → PDF renders correctly against a golden-file test (visual or text-extraction diff, whichever is more stable in CI)
- [ ] Same for DOCX
- [ ] Rendering is deterministic (same input → byte-identical or diff-stable output) so golden-file tests don't flake
- [ ] Exactly 2 templates shipped — no speculative template system

## Test plan
Golden-file tests in CI (extend `packages/infrastructure/test`), run against both templates.
