# 023 — Resume import pipeline (PDF/DOCX → structured profile → confirm)

**Status:** TODO | **Milestone:** M3

## Objective
FR-2: user uploads a resume; a worker job parses it into a structured profile draft; the user reviews/edits/confirms before it's committed as their `CareerProfile`. This is the milestone's headline accuracy-gated feature (**≥90% field accuracy** per roadmap acceptance).

## Files affected
- `packages/application/src/profile/commands/import-resume.ts` (enqueues parse job, mirrors `embed-job-posting.ts`'s outbox→worker shape)
- `apps/worker/src/handlers/parse-resume.handler.ts`
- `packages/infrastructure/src/parsing/{pdf,docx}-extractor.ts` (text/structure extraction — evaluate `pdf-parse`/`mammoth` or similar; pick the one with best layout fidelity, document the choice)
- `packages/application/src/profile/parsing/resume-field-mapper.ts` (extracted text → structured `ProfileSection[]` draft, LLM-assisted via existing `LlmPort`)
- `apps/api/src/routes/profile.ts` (`POST /api/profile/import` upload endpoint, `POST /api/profile/import/:draftId/confirm`)
- `docs/eval/resume-import-benchmark/` — a small fixture corpus (10-20 anonymized/synthetic resumes) with hand-labeled expected fields, plus a scoring script

## Dependencies
019, 020, 021, 010

## Acceptance criteria
- [ ] Import never silently commits — draft always requires explicit user confirmation (HITL, consistent with ADR-003's posture applied here)
- [ ] Field-accuracy scoring script reports ≥90% on the benchmark corpus, or the task is marked BLOCKED with the actual number and a named follow-up, not silently lowered
- [ ] Uses the existing `budget guard` / `ai_invocations` audit trail for any LLM calls (task 016 pattern) — no new unmetered LLM path
- [ ] Unsupported/corrupt file input fails with a typed error, not a crash

## Test plan
Unit (field mapper against fixed extracted text), integration (real worker + real Postgres, upload→parse→draft→confirm round trip), and the standalone accuracy-benchmark script run against the fixture corpus (report the number in this file's evidence section — this is a real measurement, not an assumption).
