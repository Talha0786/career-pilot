# 008 — Transactional outbox + relay (ADR-007)

**Status:** TODO | **Milestone:** M2

## Objective
Reliable domain-event delivery with no dual-write. Events are written in the aggregate's transaction; a relay publishes them to BullMQ.

## Files affected
- `packages/infrastructure/src/queue/outbox-relay.ts`
- `packages/infrastructure/src/queue/bullmq-queue.ts`
- `apps/worker/src/relay/index.ts`

## Dependencies
007

## Acceptance criteria
- [ ] Relay claims rows with `FOR UPDATE SKIP LOCKED` — safe with N concurrent relays
- [ ] Publishes to BullMQ, then marks `published_at`; failures increment `attempts` and record `last_error`
- [ ] Delivery is **at-least-once**; duplicates are expected and must be absorbed by idempotent handlers
- [ ] Backlog (`published_at IS NULL`, `attempts > N`) is exposed on `/admin/status`
- [ ] Poll loop is graceful-shutdown safe (finishes in-flight batch, then exits)

## Test plan
Integration: 3 concurrent relays over 500 events → every event published **exactly once** (SKIP LOCKED holds).
Chaos: `SIGKILL` the relay mid-batch, restart → zero lost events; duplicates permitted but handler must dedupe (see 010).
