# 042 — Intelligence eval harness (nightly CI gate)

**Status:** TODO | **Milestone:** M5

## Objective
Implement `docs/06-agent-design.md` §6: a golden-set eval harness tracking parse field-accuracy, mapper P0 accuracy, tailoring unsupported-claim rate, and rubric-score correlation with human-labeled samples, run nightly in CI (not per-PR — cost) against pinned models. This is the roadmap's explicit M5 acceptance criterion ("eval gates from agent-design §6 pass") and the regression gate for the anti-hallucination contract ("0 unsupported claims surviving verification").

## Files affected
- `packages/intelligence-evals/` (new workspace package) — `package.json`, `tsconfig.json`, added to `pnpm-workspace.yaml`'s `packages:` list (mirror how `docs/eval` was formalized during the M3/M4 merge fixes — but this time give it a proper `package.json` from the start, not a bolt-on fix).
- `packages/intelligence-evals/fixtures/` — golden (profile, job-description) pairs, including **deliberate trap prompts** per docs/06 §4.5 (a JD demanding a certification the profile lacks — the eval must prove the tailoring pipeline does NOT fabricate it).
- `packages/intelligence-evals/src/run-tailoring-eval.ts` — runs 039+040 against fixtures, asserts 0 unsupported claims survive verification on the trap-prompt fixtures specifically (this is the hard gate).
- `packages/intelligence-evals/src/run-matching-eval.ts` — runs 038 against fixtures with human-labeled expected scores, computes correlation (Pearson or Spearman — pick one, document the choice), tracks over time (append results to a checked-in or artifact-uploaded results log — decide storage: simplest is a timestamped JSON written to a CI artifact, per-run, not a growing checked-in file).
- `packages/intelligence-evals/src/run-resume-import-eval.ts` — migrate the existing `docs/eval/resume-import-benchmark/score.ts` logic here (or have this package import/wrap it) so all M5+ evals live in one CI-wired place; keep the 97.58%-accuracy fixture corpus from task 023, don't regress it.
- `.github/workflows/nightly.yml` — new job (alongside the existing `ollama-contract` and `connectors-live-canary` jobs) running the eval suite against pinned models, failing the workflow (not `continue-on-error`, unlike the live-canary job — this one is a real regression gate) if the tailoring trap-prompt gate or a defined accuracy threshold regresses.
- `packages/intelligence-evals/README.md` (or inline in package) — document how to add a new fixture, what "pinned models" means here and where the pin lives.

## Dependencies
- 038 (matching), 039 (tailoring), 040 (claim verification) — the eval harness runs all three pipelines.

## Acceptance criteria
- `pnpm --filter @careerpilot/intelligence-evals run eval` runs locally and produces a clear pass/fail summary (mirror `docs/eval/resume-import-benchmark/score.ts`'s existing PASS/BELOW-GATE output style).
- Nightly CI job runs the full suite against pinned models and fails the workflow run on regression of the tailoring 0-unsupported-claims gate.
- Boundary lint passes for the new package (respect Clean Architecture import rules — check `scripts/verify-boundary-enforcement.mjs` for what it allows a workspace package outside `apps/`/`packages/{domain,application,...}` to import).
- The existing resume-import benchmark's 97.58% accuracy does not regress after the migration into this package.

## Test plan
- Run the eval suite locally against whatever model is configured (key-free Ollama default per ADR-006) and capture real output as evidence in the task's Status note — do not claim this DONE without an actual eval run's output pasted into the commit message or PR, same standard applied to task 023's 97.58% benchmark.
- Confirm the nightly workflow YAML is valid (lint/dry-run if `gh` or `act` tooling is available; otherwise a careful manual review against the existing two nightly jobs' structure).
