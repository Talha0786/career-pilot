# 009 — LLM port, adapters, budget guard

**Status:** DONE | **Milestone:** M2

## Objective
Provider-agnostic LLM access (ADR-006) that **cannot** be called without a budget check and cost accounting.

## Files affected
- `packages/infrastructure/src/llm/{openai-compat.adapter.ts,budget-guard.ts,fake.adapter.ts,index.ts}`

## Dependencies
005, 006

## Acceptance criteria
- [ ] `openai-compat` adapter covers OpenAI/Ollama/vLLM; local Ollama is the **key-free default** (ADR-006)
- [ ] `BudgetGuard` checks spend **before** dispatch; over budget → `BudgetExceeded` result, **no** network call
- [ ] Every call writes an `ai_invocations` row (tokens, cost, latency, status) — including failures
- [ ] The **raw** port is not exported from the barrel; only the guarded decorator is (bypass = lint error)
- [ ] `FakeLlm` is deterministic; the default in all unit/integration tests

## Test plan
Unit: budget at $0 blocks pre-dispatch (assert the HTTP client was never invoked).
Integration: `ai_invocations` row written on success *and* on provider error.
Nightly CI: real-Ollama contract test (not in the PR path).
