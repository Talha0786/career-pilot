# 036 ‚Äî pgvector ANN index + nearest-neighbor query

**Status:** TODO | **Milestone:** M5

## Objective
`job_postings.embedding` and `career_profiles.embedding` are both `vector(768)` columns with **no ANN index** ‚Äî the schema comment explicitly deferred this to M5 ("real cardinality needed to choose parameters sensibly"). Add the index via a new migration and add the repository query method M5's matching pipeline needs: given a profile's embedding, return the top-N nearest job postings by cosine distance (the embedding prefilter from docs/06-agent-design.md ¬ß3, "embedding prefilter caps volume" before the LLM rubric-scoring pass).

## Files affected
- `packages/infrastructure/src/db/migrations/0004_ann_index.sql` (new) ‚Äî `CREATE INDEX ... USING hnsw (embedding vector_cosine_ops)` on both `job_postings` and `career_profiles` (pgvector HNSW; use `m=16, ef_construction=64` as defaults unless the existing schema doc's design notes specify otherwise ‚Äî check `docs/02-database-design.md` for any prior guidance before picking parameters).
- `packages/application/src/ports/repositories.ts` ‚Äî extend `JobPostingRepository` with `findNearestByEmbedding(embedding: number[], opts: {limit: number, excludeStatuses?: string[]}): Promise<JobPosting[]>` (or similar ‚Äî match the existing repository method-naming style in that file).
- `packages/infrastructure/src/db/repositories/drizzle-job-posting.repository.ts` (or wherever `DrizzleJobPostingRepository` lives) ‚Äî implement via Drizzle's pgvector cosine-distance operator (`<=>`), `ORDER BY embedding <=> $1 LIMIT $2`.
- `packages/infrastructure/test/integration/` ‚Äî new or extended integration test: seed several job postings with distinct embeddings, prove nearest-neighbor ordering is correct against real Postgres+pgvector.

## Dependencies
- 035 (profile embeddings must exist for a meaningful profile‚Üíjob ANN integration test; the index migration itself has no hard dependency, but write the task as a unit so the integration test isn't testing against always-null profile embeddings).

## Acceptance criteria

- `findNearestByEmbedding` returns results ordered by ascending cosine distance, respects `limit`.
- No regression to existing job-postings integration tests.
